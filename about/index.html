基于Pytest+MCP Server的接口自动化测试框架

GitHub链接

技术栈： Python+Pytest+Requests+YAML+Allure+Logging+GitHub Actions+Chrome DevTools MCP

  • 测试框架设计：采用 API Object 分层架构,设计了配置、数据、用例、逻辑的四层解耦,并采用基于YAML的数据驱动引擎
  • 自动化鉴权：利用Fixture机制实现全局登录,结合中间文件存储 Token,自动在封装层注入 Header 解决接口依赖

  • 并发测试：基于pytest-xdist实现多进程并发执行,利用FileLock解决多进程场景中间数据文件的资源竞争问题
  • CI/CD集成：基于 GitHub Actions 搭建自动化流水线,代码提交即触发测试且自动生成发布 Allure 报告并消息通知
  • MCP工具二开：基于Chrome DevTools MCP进行二次开发，新增网络请求导出工具，可自动捕获浏览器fetch/xhr请求
    并转换为框架所需的YAML测试数据格式，实现在AI驱动 UI 自动化测试期间，同步抓取并清洗页面接口请求

基于LightRAG的蜂巢AI校园生活助手 | app开发

GitHub链接

技术栈： FastAPI+UniAPP+Vue+LightRAG+Docker+PostgreSQL+PGvector+DeepSeek API+阿里云云端+阿里云ECS

  • 框架设计：FastAPI 构建 RESTful API,移动端使用 UniAPP 跨平台框架,通过 uni.request 封装统一的 API 调用层
  • RAG：利用DeepSeek对知识库进行实体和关系的抽取,知识图谱存储在本地,通过 Docker 卷持久化,向量/KV/文档状态存
    入PostgreSQL的PGvector, text-embedding-v4作为向量模型.qwen3-rerank作为排序模型,qwen3-max作为文本生成模型
  • 多模态交互：利用qwen-audio-asr模型来支持语音识别,qwen3-vl-flash模型支持OCR识别
  • 项目部署：Docker Compose实现本地一键部署,阿里云 ECS 生产环境部署.Hbuilder云打包APK,已上传至Releases
  • 测试：利用JMeter进行压力测试,重点监控检索阶段与阿里百炼 API 响应的延迟分布;通过分析测试结果,针对性调整了
    Docker容器的并发控制参数MAX_ASYNC,在确保系统高负载稳定性的同时将平均响应延迟降低了15%
