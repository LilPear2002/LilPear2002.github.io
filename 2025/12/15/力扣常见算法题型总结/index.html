越短越合法

一般要写 ans += right - left + 1。

内层循环结束后，[left,right] 这个子数组是满足题目要求的。由于子数组越短，越能满足题目要求，所以除了 [left,right]，还有 [left+1,right],[left+2,right],…,[right,right] 都是满足要求的。也就是说，当右端点固定在 right 时，左端点在 left,left+1,left+2,…,right 的所有子数组都是满足要求的，这一共有 right−left+1 个。

二分答案

当你发现一个问题的答案具有单调性（即：如果x满足条件，那么所有大于x或小于x的值也一定满足/不满足条件），你就可以不再直接寻找答案，而是先“猜”一个答案，然后验证这个答案是否合法。

求最小

对于求最小的题目，开区间二分的写法，为什么最终返回的是 right，而不是别的数？在初始化（循环之前）、循环中、循环结束后，都时时刻刻保证 check(right) == true 和 check(left) == false，这就叫循环不变量。根据循环不变量，循环结束时 left+1=right，那么 right 就是最小的满足要求的数（因为再 −1 就不满足要求了），所以答案是 right。

做这道题时的疑问：为什么代码对left的初始化这么敏感，在我的理解中，以开区间为例，区间为（left，right），left应该足够小，小到保证在区间左边的取值，也就是小于等于left的值都无法满足条件，而right应该足够大，大到在区间右边的值均可以满足条件即可，也就是说，满足这两个条件的初始化，只会影响运行的速度，而不会出现案例无法通过的情况，但是我将left初始化为1（肯定无法满足判断条件）,或者改为left = max(weights) - 2，就会出现无法通过的案例，而改right=sum(weights)+1就不影响

AI回答：在 1011. 在 D 天内送达包裹的能力 这道题中，你遇到的问题其实不在于二分算法本身，而在于你的 check 函数中隐含的一个逻辑前提。

为什么 left = max(weights) - 1 是安全的？

因为当 mid >= max(weights) 时，y = mid 之后，执行 y -= x 永远不会让 y 变成负数（或者说，单个货物的重量永远不会超过一天的总容量）。

为什么 left = 1 会出错？

如果 weights = [10, 20]，而二分过程中某次 mid = 5：

  1. 遍历到 x = 10。
  2. 10 > 5 成立，cnt 变为 2，y 重置为 5。
  3. 关键点： 执行 y -= x 即 5 - 10 = -5。此时剩余容量 y 变成了负数！
  4. 下一次循环，即便 x 很小，也会因为 x > -5 再次触发 cnt += 1。

本质原因： 你的 check 函数逻辑默认了 “单件货物的重量 x 绝对不会超过运送能力 mid”。如果 mid 小于数组中的最大值，这个 check 函数计算出的 cnt 就失去了物理意义，无法准确代表所需天数。

两种解决方案

方案一：修正 check 函数（不推荐）

让 check 函数能处理 mid 极小的情况。如果某个货物重量 x > mid，直接返回 False（表示这种运载能力绝对不可能完成任务）。

这样修改后，你把 left 初始化为 0 或 1 就都没问题了。

方案二：维持现状，确保 left 的含义（推荐）

在二分答案中，定义好 “解的可能范围” 非常重要。

  • 下界： 最沉的那件货必须能装上船，所以运载能力至少要 max(weights)。
  • 上界： 一天全部送完，所以运载能力至多 sum(weights)。

所以，将 left 设置为 max(weights) - 1 不仅仅是为了速度，更是为了保证进入 check 函数的 mid 永远是一个逻辑上合理的“候选答案”。

结论： 在写二分答案时，check 函数的逻辑严密性 与 初始边界的选择 是相辅相成的。如果 check 写得比较“脆”（依赖某些前提），边界就必须定得更精准。

求最大

请注意「求最小」和「求最大」在二分写法上的区别。

「求最小」和二分查找求「排序数组中某元素的第一个位置」是类似的，按照红蓝染色法，左边是不满足要求的（红色），右边则是满足要求的（蓝色）。

「求最大」的题目则相反，左边是满足要求的（蓝色），右边是不满足要求的（红色）。这会导致二分写法和上面的「求最小」有一些区别。

以开区间二分为例：

  • 求最小：check(mid) == true 时更新 right = mid，反之更新 left = mid，最后返回 right。
  • 求最大：check(mid) == true 时更新 left = mid，反之更新 right = mid，最后返回 left。

对于开区间写法，简单来说 check(mid) == true 时更新的是谁，最后就返回谁。相比其他二分写法，开区间写法不需要思考加一减一等细节，推荐使用开区间写二分。

java中同上

python中也有deque双端队列。deque支持多种方法，允许在两端添加（append和appendleft）或移除（pop和popleft）元素。它还提供了extend和extendleft方法来从可迭代对象中添加多个元素。此外，deque还支持rotate方法，可以将队列中的元素向右或向左旋转
