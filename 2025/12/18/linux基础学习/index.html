linux基础学习

Linux介绍

简单说，Linux 和平时用的 Windows、macOS 一样，都是操作系统（Operating System）。

但它有两个核心特点，注定让它成为了服务器领域的霸主：

  1. 开源免费：源代码是公开的，任何人都可以修改。
  2. 极度稳定：它可以连续运行几年不关机、不重启，而且效率极高。

在 Windows/Mac 电脑上写代码、写文档（这是工作环境）；但测试的代码、数据库、服务，通常都运行在远端的 Linux 服务器上（这是生产/测试环境）。

可以把 Linux 系统想象成一家餐厅：

  • 硬件（Hardware） ➜ 厨房的食材和灶具
    • 这是底层的 CPU、内存、硬盘。
  • 内核（Kernel） ➜ 大厨
    • 这是 Linux 的核心。只有大厨（内核）有权直接操作灶具（硬件），别人不能乱动。它负责管理内存、CPU 调度。
  • Shell（终端/命令行） ➜ 服务员
    • 这就是以后要面对的那个“黑底白字的窗口”。
    • 顾客不能直接进厨房炒菜，必须把需求告诉服务员（输入命令），服务员传达给大厨（内核），大厨做好后，服务员再显示结果。
  • 应用程序（Applications） ➜ 菜品
    • 在上面跑的 MySQL 数据库、Tomcat、Nginx 或者写的 Python 自动化脚本。

Linux 发行版（Distributions）

Ubuntu、CentOS、RedHat、Debian… 其实，Linux 内核 只有一个。但是不同的公司为了方便用户使用，把内核 + 各种常用软件 + 安装工具打包在一起，就变成了发行版。

  • CentOS / RedHat：企业级服务器**最常用的版本。特点是稳，但软件版本比较老。
  • Ubuntu：开发者和 AI 领域最喜欢的。界面友好，软件比较新。
  • Kali Linux：黑客和安全测试人员专用的（如果你做安全测试会用到）。

命令是通用的。学会了 CentOS，Ubuntu 也能上手，反之亦然。

为什么测开不用图形界面（GUI）

在 Linux 服务器上，通常通过 SSH（远程连接） 对话。

  1. 省资源：图形界面非常吃内存和 CPU。服务器资源要留给跑代码，而不是用来渲染漂亮的窗口。
  2. 效率：
    • 想删除 1000 个特定的日志文件。
    • Windows：按住 Ctrl 选半天，或者框选。
    • Linux：一行代码 rm *.log，0.1 秒搞定。
  3. 自动化：测开的核心是自动化。图形界面的操作很难写进脚本里，但命令行（Shell 脚本）天生就是为了自动化而生的。

问题：为什么说 Linux 没 GUI，但有时候在双系统/租云服务器时又能看到界面？

Linux内核 本身只是一个命令行（文本界面）的核心。它的主要职责是管理硬件和资源（进程、内存、设备等），不负责绘制窗口、按钮、菜单或处理鼠标点击。而GUI是它们在Linux内核之上，叠加了一整套庞大的“图形化软件栈”。这套软件栈不是内核的一部分，而是运行在内核之上的应用程序和服务。简单来说，图形界面只是一个普通的软件。

云服务器

简单理解为：一台位于远程数据中心、24小时不断电、通过网络租给你用的高性能电脑。

在互联网公司（比如你要去的实习岗位），所有的代码、数据库、测试环境，都不会跑在开发人员自己的笔记本上，而是跑在云服务器上。

  • 随时随地访问：合上笔记本回家了，服务器还在运行自动化测试脚本。
  • 公网 IP：云服务器有一个公网 IP 地址。在上面部署一个网站，全世界的人都能通过浏览器访问它。
  • 弹性：今天测试压力小，租个 2核4G 的；明天搞双十一压测，一键升级成 64核128G 的。

2. 打个比方

  • 虚拟机/WSL（本地环境）：就像在自己家里买了一台跑步机。如果不维护（关机/重装系统），它就停了。
  • 云服务器：就像办了一张健身房年卡。场地、设备人家都维护好了，只需要去“用”就行，而且那是公共场所（公网），可以约朋友（用户）一起来。

基础命令

目录相关命令

导航

Linux 是一个倒挂的大树，树根是 /。

  • pwd (Print Working Directory)：显示当前路径。
    • 输入 pwd 回车。应该在 /root
  • ls (List)：看看这里有什么。
    • 试一试：输入 ls，或者更常用的 ls -l (显示详细信息)。
  • cd (Change Directory)：传送门。
    • 试一试：
      1. cd / (回到根目录，这里是系统的起点)
      2. cd ~ (波浪号代表回家。无你在哪里，一键回到家目录)

文件操作

创建文件

  • mkdir (Make Directory)：建文件夹。
    • 试一试：mkdir my_test_project
  • touch：建空文件（或者更新文件时间戳）。
    • 试一试：cd my_test_project (进入刚才建的目录)，然后 touch readme.txt
  • cp (Copy)：复制（备份神器）。
    • 试一试：cp readme.txt readme.txt.bak
    • 场景：修改生产环境配置前，必须先 cp 备份一份，防止改挂了回不去。
  • mv (Move)：移动，或者重命名。
    • 试一试：mv readme.txt config.txt (把 readme.txt 改名为 config.txt)
    • mv和cp命令在移动或复制过程中都可以重命名
  • rm (Remove)：删除（慎用！）。
    • 试一试：rm readme.txt.bak (删除刚才那个备份文件)
    • 高危警告：rm -rf 目录名 是强制删除目录及其下所有东西。千万不要手滑敲成 rm -rf /，那样服务器就“自杀”了。

查找文件

文件编辑

在 Linux 上用 Vim。 Vim 有两个模式

  1. 命令模式（默认）：键盘是用来发号施令的（复制、粘贴、保存），不能打字。
  2. 编辑模式：这才是打字的。

⚡️ 极速上手三板斧：

  1. 打开文件：vim config.txt
  2. 开始打字：按一下 i 键（Insert）。（发现左下角变成了 -- INSERT --，现在可以像记事本一样打字了）。
  3. 保存退出：
    • 按一下 Esc 键（退出编辑模式，左下角 INSERT 消失）。
    • 输入 :wq
    • 回车。
    • 解释：:是指令开始，w=write(保存)，q=quit(退出)。

如果不小心搞乱了想强制退出（不保存）：按 Esc，输入 :q! 然后回车。

文件解压缩

tar 是 Linux 最经典的打包命令，通常打包出来的文件叫 .tar.gz。只需要记住两句口诀：

  • 打包（压缩） ➜ czvf
    • 命令：tar -czvf my_logs.tar.gz ./logs
    • 记忆法：Create (创建) Zipped (压缩) Verbose (显示过程) File (档案)。
    • 意思：把当前目录下的 logs 文件夹，打包成 my_logs.tar.gz。
  • 解包（解压） ➜ xzvf
    • 命令：tar -xzvf my_logs.tar.gz
    • 记忆法：Xtract (提取/解压)。把 C 换成 X 就行了。

权限管理

你写了一个脚本，结果运行报错 Permission denied？这是权限问题。

输入 ls -l (或者 ll)，你会看到类似 -rw-r–r– 的东西。

  • r = Read (读)
  • w = Write (写)
  • x = Execute (执行/运行)

关键命令：chmod (Change Mode)

  • 场景：你写了一个自动化脚本 run.sh，想运行它。
  • 试一试：
    1. touch run.sh
    2. ls -l run.sh (注意看，它是没有 x 权限的，通常是白色)
    3. chmod +x run.sh (赋予执行权限)
    4. ls -l run.sh (注意看，它变色了！变成了可执行文件)

进程与端口管理

  • “我部署了服务，但网页打不开，是没启动吗？”
  • “我想启动一个新的测试环境，系统提示端口被占用（Address already in use）。”
  • “服务器 CPU 突然飙到 100%，是谁在搞鬼？”

解决这些问题，需要掌握这套“查进程 -> 杀进程 -> 查端口”的连招。

查看进程

在 Windows 任务管理器里可以看到运行的所有软件。在 Linux 里，用 ps (Process Status)。

核心命令：ps -ef

这会列出系统当前运行的所有进程。但因为进程太多，直接敲这个命令会刷屏。所以配合 grep：

查看 python 程序有没有在运行。

会看到类似这样的很多行：

root 1234 1 0 10:00 ? 00:00:00 python3 server.py

  • root: 谁启动的
  • 1234 (PID): 身份证号 (Process ID)。这是操作它的唯一凭证，后面傻进程全靠它。
  • python3 server.py: 启动命令是什么？

💡 注意：通常会看到两行，其中一行是 grep python 本身，那是你刚才敲的命令，忽略它。

进阶命令：top

如果想看“谁最吃 CPU”或者“内存还剩多少”，输入 top。

  • 这就相当于 Windows 任务管理器的“性能”页。
  • 按 q 退出。

结束进程

当某个服务卡死了，或者想重启它，需要先把它“杀掉”。

核心命令：kill

  • 温柔一刀：kill PID
    • 例如：kill 1234
    • 这是发送信号让程序“收拾收拾自己退出”，大部分正常程序会听话。
  • 强制处决：kill -9 PID
    • 例如：kill -9 1234
    • -9 代表“强制杀死”。不管程序正在干嘛，直接从内存里抹去。当程序死锁无响应时，用这个。

查看端口

有时候进程在运行，但网页打不开，可能是**端口（Port）**没对上，或者被防火墙挡了。

核心命令：netstat 或 ss

注：如果提示 netstat: command not found，请先运行 apt install net-tools 安装它。

想知道 8080 端口现在是被谁占用的

*(记忆口诀：nlp = Numeric(数字显示) Listening(监听中) Program(显示程序名))*

输出解读：tcp 0 0 0.0.0.0:8080 0.0.0.0:* LISTEN 5678/java

  • 说明 8080 端口正在被 PID 为 5678 的 java 程序占用。
  • 如果想启动新程序也用 8080，就得先 kill 5678。

监控与分析

实时监控 (tail -f)

场景：在前端点一个按钮（比如“提交订单”），页面转圈圈没反应。需要立刻知道后台此刻报没报错。

这时候用 cat 是不行的，因为文件是静态的。需要动态地盯着它。

  • 命令：tail -f 文件名
    • -f = follow（跟随）。
    • 效果：屏幕会卡住，一旦日志文件有新内容写入，屏幕上会立刻滚动显示出来。就像你看直播弹幕一样。
  • 退出：按 Ctrl + C。

多维过滤 (Advanced grep)

第一阶段学了基础的 grep，但实战中情况更复杂。

  • 场景 A：不仅想看 Error，还想看 Warning
    • 命令：grep -E "Error|Warning" server.log
    • 解释：-E 开启扩展正则，| 代表“或者”。
  • 场景 B：报错太多了，想统计一下到底报了多少次
    • 命令：grep "Error" server.log | wc -l
    • 解释：| 是管道符（把前一个命令的结果传给后一个命令）；wc -l (Word Count -lines) 是之前学的数行数。
    • 结果：直接输出一个数字，比如 520。
